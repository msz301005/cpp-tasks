# A. String [C++ fall 2023]

| | |
|---|---|
| Ограничение времени | 1 секунда |
| Ограничение памяти | 256Mb |
| Ввод | стандартный ввод или input.txt |
| Вывод | стандартный вывод или output.txt |

В этой задаче запрещено пользоваться какими-либо стандартными контейнерами. Выделения памяти нужно делать вручную. Разрешается подключать только `<iostream>` и `<cstring>` (можно подключить `<algorithm>` для использования `std::fill` и `std::copy`).

Напишите класс String - упрощенный аналог библиотечного `std::string`. Необходимо поддержать следующий функционал:

- Конструктор от C-style строки (`const char*`)
- Конструктор от двух параметров: число n и символ c - создает строку из n копий символа c.
- Конструктор по умолчанию, копирующий конструктор и оператор присваивания. Оператор присваивания копией должен корректно обрабатывать присваивание самому себе.
- Оператор `==`, позволяющий проверять строки на совпадение, оператор `!=`, а также операторы `< <= > >=` для лексикографического сравнения строк.
- Квадратные скобки, корректно работающие как для константных, так и для неконстантных строк.
- Метод `length()`, возвращающий размер строки. А также методы `size()` и `capacity()`, возвращающие соответственно размер и текущую емкость. Терминирующий символ 0 ни там, ни там не учитывается.
- Методы `push_back(char)` и `pop_back()`. Амортизированное время работы `push_back` должно быть O(1). Метод `pop_back` должен работать за гарантированное O(1), то есть не перевыделять память.
- Методы `front()` и `back()`, возвращающие ссылку на начальный и последний символы строки соответственно. Должны корректно работать и для константных, и для неконстантных.
- Оператор `+=`, позволяющий добавить к строке символ или другую строку. Добавление символа должно работать за амортизированное O(1).
- Оператор `+`, позволяющий складывать строки с символами, символы со строками и строки со строками.
- Методы `find(substring)` и `rfind(substring)`, позволяющие найти самое левое и самое правое вхождение подстроки substring в данную строку. Возвращают индекс найденного вхождения. Если вхождение не найдено, нужно вернуть `length()` (фиктивный индекс).
- `substr(start, count)`, возвращающий подстроку начиная с индекса start и состоящую из count символов.
- Метод `empty()`, проверяющий, пуста ли строка. Метод `clear()`, позволяющий очистить строку. В методе clear делается только обнуление размера, но не освобождение памяти.
- Метод `shrink_to_fit()`, освобождающий лишнюю неиспользуемую память.
- Операторы вывода в поток `<<` и ввода из потока `>>`.
- Метод `data()`, возвращающий указатель на начало строки. Корректно должен работать как для константных, так и для неконстантных строк.

**Дополнительные замечания:**

- Строка должна заканчиваться символом с кодом 0. (Стандартный `std::string` тоже обладает этим свойством.) Это нужно, чтобы в случае использования `data()` можно было корректно работать с полученной сишной строкой.
- Количество вызовов `new` нужно стараться свести к минимуму. В частности, в операторе `+=` нельзя добавлять символ за раз, надо сразу выделять память на всю добавляемую строку. Аналогично в методе `substr`. При операторе присваивания, если текущая строка длиннее присваиваемой строки, обойтись без перевыделения памяти.
- Для заполнения строки символами, а также для копирования строк не надо использовать циклы с полиномиальным присваиванием. Надо использовать `memset` и `memcpy`, а еще лучше - `std::fill` и `std::copy`.

В вашем файле должна отсутствовать функция `main()`, а сам файл должен называться `string.h`. В качестве компилятора необходимо указывать `MakeCC ++17`. Ваш код будет вставлен посредством `#include "string.h"` в программу, содержащую тесты; вследствие этого код необходимо отправлять в файле со строго соответствующим именем!

# C. Geometry [C++ fall 2023]

| | |
|---|---|
| Ограничение времени | 2 секунды |
| Ограничение памяти | 64Mb |
| Ввод | стандартный ввод или input.txt |
| Вывод | стандартный вывод или output.txt |

Напишите иерархию классов для работы с геометрическими фигурами на плоскости.

- **Структура Point** - точка на плоскости. Точку можно задать двумя числами типа double. Должны быть открыты поля x и y. Точки можно сравнивать операторами `==` и `!=`.
- **Класс Line** - прямая. Прямую можно задать двумя точками, двумя коэффициентами и сдвигом (угловой коэффициент и сдвиг), либо точкой и числом (угловой коэффициент). Линии можно сравнивать операторами `==` и `!=`.
- **Абстрактный класс Shape** - фигура.
- **Класс Polygon** - многоугольник. Многоугольник - частный случай фигуры. У многоугольника можно спросить `verticesCount()` - количество вершин - и `std::vector<Point>` `getVertices()` - сами вершины без возможности изменения. Можно спросить `isConvex()` - выпуклый ли. Можно сконструировать из вектора точек вершин в порядке обхода. Можно сконструировать многоугольник из точек, передаваемых в качестве параметров через запятую (то, неуказанное число аргументов). Для простоты будем считать, что многоугольники самопересекающихся никогда не возникает (гарантируется, что в тесты подобные будут отсутствовать), кроме того, считаем, что последовательные вершины многоугольника никогда не лежат на одной прямой.
- **Класс Ellipse** - эллипс. Эллипс - частный случай фигуры. У многоугольников можно спросить `std::pair<Point, Point> focuses()` - его фокусы; `std::pair<Line, Line> directrices()` - пару его директрис; фокальный `eccentricity()` - его эксцентриситет, `Point center()` - его центр. Эллипс можно сконструировать из двух фокусов и суммы расстояний от точки до них.
- **Класс Circle** - круг. Круг - частный случай эллипса. У круга можно спросить `radius()` - радиус. Круг можно задать точкой и числом (центр и радиус).
- **Класс Rectangle** - прямоугольник. Прямоугольник - частный случай многоугольника. У прямоугольника можно спросить `Point center()` - его центр; `std::pair<Line, Line> diagonals()` - пару его диагоналей. Прямоугольник можно сконструировать из двух (противоположных вершин) и числа (отношение сторон), причем из двух таких прямоугольников выбирается тот, у которого более короткая сторона расположена по левую сторону от диагонали, если смотреть от заданной точки в направлении второй.
- **Класс Square** - квадрат. Квадрат - частный случай прямоугольника. У квадрата можно спросить `Circle circumscribedCircle()`, `Circle inscribedCircle()`. Квадрат можно задать двумя точками - противоположными вершинами.
- **Класс Triangle** - треугольник. Треугольник - частный случай многоугольника. У треугольника можно спросить `Circle circumscribedCircle()`, `Circle inscribedCircle()`, `Point centroid()` - точку пересечения медиан, `Point orthocenter()` - ортоцентр, `Line EulerLine()` - прямую Эйлера, `Circle ninePointsCircle()` - окружность девяти точек.

У любой фигуры можно спросить:

- `double perimeter()` - периметр;
- `double area()` - площадь;
- `bool operator==(const Shape& another)` - совпадает эта фигура с другой как множество точек. (В частности, треугольник ABC равен треугольнику BCA.)
- `bool isCongruentTo(const Shape& another)` - равна ли эта фигура другой в геометрическом смысле, то есть можно ли совместить эти две фигуры на плоскости движением. (это отображение плоскости на себя, сохраняющее расстояние).
- `bool isSimilarTo(const Shape& another)` - подобна ли эта фигура другой, то есть можно ли перевести одну фигуру в другую преобразованием подобия. (Определение преобразования подобия, если не знает, можно посмотреть в Википедии.)
- `bool containsPoint(const Point& point)` - находится ли точка внутри фигуры.

Фигуры не обязаны иметь одинаковый тип, чтобы считаться равными, конгруэнтными или подобными! Любую фигуру нужно сравнить с любой другой и получить правильный ответ, независимо от их истинных типов.

С любой фигурой нужно сделать:

- `rotate(Point& center, double angle)` - поворот на угол (в градусах, против часовой стрелки) относительно точки;
- `reflect(const Point& center)` - симметрию относительно точки;
- `reflect(const Line& axis)` - симметрию относительно прямой;
- `scale(Point& center, double coefficient)` - гомотетию с коэффициентом coefficient и центром center.

В вашем файле должна отсутствовать функция `main()`, а сам файл должен называться `geometry.h`. В качестве компилятора необходимо выбирать Make. Ваш код будет вставлен посредством `#include` в программу, содержащую тесты.

# D. Matrix [C++ fall 2023]

| | |
|---|---|
| Ограничение времени | 30 секунд |
| Ограничение памяти | 64Mb |
| Ввод | стандартный ввод или input.txt |
| Вывод | стандартный вывод или output.txt |

Баллы за эту задачу можно получить только при прохождении тестов по задачам BigInteger и Rational.

Эту задачу можно сдавать на C++23 (та, в которой её поддерживает clang++ версии 16). В частности, можно использовать оператор `<=>` с двумя аргументами. Если вы сомневаетесь так делать, то в своём коде указывайте `#define CPP23 1`. Тогда с тестов ваши матрицы будут делаться так [i,j], а так [i][j].

Написать шаблонный класс `Residue<size_t N>` - кольцо вычетов по модулю N (которое, как известно, в случае простого N является полем). Должны поддерживаться арифметические операции: сложение, вычитание, умножение, деление и деление по остатку (`Residue<N>` должно иметь конструктор от целого типа) и питг взятия обратного элемента по N в мультипликативном смысле). Также необходимо привести `Residue<N>` к int (в преобразовании должна быть однозначность) для перебора приводить к соответствующим значениям. Проверка числа N на простоту должна выполняться compile-time, то есть за O(sqrt(N)).

После этого, используя ранее написанный класс рациональных чисел, класс матрица с шаблонными параметрами `size_t M, size_t N, typename Field=Rational`. (По умолчанию рациональных чисел, но можно приводить с другим полем.) Первым параметром идёт число строк, вторым - число столбцов.

Конструктор матрицы для возвращать матрицы из одних нулей (или единичную, если M == N, то это квадратная матрица не компилируется).

Матрицы должны поддерживать следующие операции:

- Сложение, вычитание, операторы `+=`, `-=`. Сложение и вычитание матриц несоответствующих размеров не должно компилироваться.
- Умножение на число. Матрица. Перемножить матрицы несоответствующих размеров должна приводить к ошибке компиляции.
- Умножение двух матриц.
- Метод `det()`, возвращающий определитель матрицы за O(N^3). Взятие определителя матрицы, у которой M != N, не должно, конечно же, компилироваться.
- Метод `transposed()`, возвращающий транспонированную матрицу.
- Метод `rank()` - вычисление ранга матрицы, которое не должно предполагать наличие поля. Вычисление ранга должно работать за O(N^3).
- Методы `inverted()` и `invert()` - вернуть обратную матрицу и обратить данную матрицу. Попытка обратить матрицу, у которой M != N, должна приводить к ошибке компиляции.
- Метод `trace()` - вычислить след матрицы. Вычисление следа от неквадратной матрицы не должно компилироваться.
- Метод `getRow(unsigned)` и `getColumn(unsigned)`, возвращающие std::array со значениями.
- К матрицам должен быть доступ применим оператор [], причём его следует применять как для неконстантных, так и для константных матриц. В первом случае элементы матрицы должны быть доступны как значения.

Другие способы изменения содержимого матрицы, кроме описанных выше, должны быть отсутствовать. Задавать матрицу непосредственно методом ввода для двоичных или каких-либо алгебраических операций не может быть предоставлено делать для другой работы, если по показанию и синтаксисе типа будет допустимо обозначить всего с одним обязательным шаблонным параметром: `SquareMatrix<N>`.

Помимо класса Matrix, реализуйте также шаблонное выражение matrix expressions, как описано в статье [ru.wikipedia.org/wiki/Expression_templates].

А именно, определите шаблонный класс `MatrixExpression`, от которого унаследуйте с помощью паттерна CRTP как класс Matrix, а также классы `MatrixSum`, `MatrixDifference` и `MatrixProduct`. Оператор сложения матриц должен конструировать объект `MatrixSum`, вычитания - `MatrixDifference`, а оператор умножения - конструировать объект `MatrixProduct`. Объект Matrix должен иметь конструктор от любого MatrixExpression, из которого вычисляется Matrix, то есть все элементы матрицы должны вычисляться заранее.

# E. Deque [C++ spring 2024]

| | |
|---|---|
| Ограничение времени | 2 секунды |
| Ограничение памяти | 64Mb |
| Ввод | стандартный ввод или input.txt |
| Вывод | стандартный вывод или output.txt |

Если понадобится, в этой задаче можно использовать `std::vector` и `std::array`. Другие стандартные контейнеры использовать нельзя. Необходимо написать шаблонный класс `Deque<T>` - сильно упрощенный аналог класса `std::deque<T>`. Тип T не обязан иметь конструктор по умолчанию, чтобы его можно было хранить в Deque.

Ваш Deque должен обладать следующей функциональностью:

- Конструкторы: по умолчанию, конструктор копирования, конструктор от int, конструктор от (int, const T&). Оператор присваивания deque.
- Метод `size()`, возвращающий текущий размер контейнера.
- Обращение по индексу: квадратными скобками (без проверок на границу) и `at` (кидающее `std::out_of_range`). Должно работать за гарантированное O(1).
- Методы `push_back`, `pop_back`, `push_front`, `pop_front`. Должны работать за амортизированное O(1).
- Должен внутренний тип `iterator` (с маленькой буквы, в лучших традициях STL). Этот тип, помимо очевидного, должен поддерживать:
  - Инкремент, декремент
  - Сложение с целыми числами
  - Сравнение `< <= > >= == !=`
  - Взятие разности двух итераторов
  - Разыменование (унарная звездочка), результатом является T&
  - Оператор `->`, результатом является T*
- Также должен быть константный итератор `const_iterator`. Отличие его от обычного iterator в том, что он не позволяет менять лежащий под ним элемент. Конверсия (в том числе неявная) неконстантного итератора в константный, обратно - нет. Реализовать константный итератор надо так, чтобы его код не являлся копипастой кода обычного итератора.
- Также должны быть конструктор копирования и константный итератор, реализовать константный итератор надо так, чтобы код не являлся копипастой обычного итератора.
- Начало и конец, возвращающие неконстантные итераторы на начало и на "элемент после конца" контейнера соответственно. Если сам контейнер константный, то методы begin и end тоже возвращают константные итераторы.
- Декремент end должен давать итератор на последний элемент, вычитание целых чисел из end должно корректно работать и давать итераторы на соответствующие элементы.
- reverse-итераторы, а также методы `rbegin`, `rend`, `crbegin`, `crend`.
- Метод `insert(iterator, const T&)`, делающий вставку в контейнер по итератору. Все элементы справа сдвигаются на один вправо, вставка работает линейное время.
- Метод `erase(iterator)`, удаляющий элемент из контейнера по итератору. Все элементы справа сдвигаются на один влево, удаление работает линейное время.
- **Все методы вашего Deque (кроме insert и erase по итератору) должны быть строго безопасны относительно исключений.** Это значит, что в случае исключения при работе какого-либо метода дека, последний должен вернуться в исходное состояние, которое было до начала выполнения метода, и пробросить исключение к вызывающему коду.

В вашем файле должна отсутствовать функция `main`, а сам файл должен называться `deque.h`. Ваш код будет включен посредством `include` в программу, содержащую тесты.

# F. List and StackAllocator [C++ spring 2024]

| | |
|---|---|
| Ограничение времени | 10 секунд |

| Ввод | стандартный ввод или input.txt |

**Предупреждение:** формально следующие задачи не зависят от того, есть для получения баллов по любой из остальных задач, чтобы задача была сдана. Однако, код, написанный в этой задаче, будет предполагаться для переиспользования в следующей задаче.

В этой задаче вам предлагается научиться пользоваться нестандартными аллокаторами, а также разработать с устройством контейнера list на их основе, чтобы в итоге, когда мы получим нестандартный аллокатор может давать выигрыш в производительности по сравнению со стандартным.

## Часть 1

Напишите класс `StackAllocator<typename T, size_t N>`, который позволял бы создавать стандартные контейнеры со всего одного обращения к динамической памяти. Для этого вам пригодится класс `StackStorage<size_t N>`, в котором будет храниться выделенная память на стеке. Класс StackStorage не должен быть копируемым, а разработка с публикация класса аллокатора должна использоваться как аллокатор для стандартных контейнеров.

Класс StackAllocator должен удовлетворять требованиям, описанным на странице [en.cppreference.com/w/cpp/named_req/Allocator]. Он должен быть STL-совместим, что позволяло использоваться в качестве аллокатора для стандартных контейнеров. В частности, должны быть определены:

- Конструктор по умолчанию, конструктор копии, деструктор, оператор присваивания
- Внутренний тип `value_type`
- Методы `allocate, deallocate`
- Метод `select_on_container_copy_construction`, если логика вашего аллокатора этого потребует.

Пример того, как может использоваться StackAllocator:

```cpp
int main() {
    StackStorage<100000> storage;
    StackAllocator<int, 100000> alloc(storage);
    std::vector<int, StackAllocator<int, 100000>> v(alloc);
    // ... useful stuff ...
}
```

Во время испытания может существовать несколько StackStorage с одним и тем же N. Разумеется, аллокаторы, построенные на разных StackStorage, должны быть отдельно считаны.

**StackAllocator должен заботиться о выравнивании объектов.** В частности, нельзя память типа int по адресу, кратному 4, чтобы double по адресу, кратному 8, и т.д.

Использование любого аллокатора со std::allocator примерно так: std::allocator придумать по умолчанию для аллокатора динамической памяти и программе не производит выигрыш.

Проверьте себя: напишите функцию, которая создаёт std::vector<int> и заполняет его случайными/добавляемыми элементами из листа. Заметьте всё насколько, если сравнить со временем работы std::vector со StackAllocator по сравнению со стандартным аллокатором.

## Часть 2

Напишите класс `List` - двусвязный список с правильным использованием аллокатора. При этом ваш List должен удовлетворять требованиям [en.cppreference.com/w/cpp/named_req/AllocatorAwareContainer], а именно:

Должны быть шаблонные параметры: Т - тип шаблона элемента и Alloc - тип используемого аллокатора (по умолчанию `std::allocator<T>`).

Должны быть определены:

- Конструкторы: без параметров; от одного числа; от числа count и T&, aллокатора; от одного aллокатора, const T&, и аллокатора. **Список должен работать без каких-либо ненужных элементов, то есть без выделения памяти под лишние объекты того типа T, если он не подходит ни под какие такие конструкторы (то есть не нужно вызывать конструктор по умолчанию для T).**
- Метод `get_allocator()`, возвращающий объект аллокатора, использующегося в данном контейнере.
- Конструктор копии, копирующий оператор присваивания
- Метод `size()`, работающий за O(1)
- Методы `push_back, push_front, pop_back, pop_front`.
- Двунаправленные итераторы, включая константные и reverse-итераторы. Также поддержка const и reverse-итераторов.
- Методы `begin, cbegin, end, cend, rbegin, rend, crbegin, crend`.
- Методы `insert(const_iterator, const T&)` и `erase(const_iterator)`. Причём для добавления и удаления одиночных элементов в список.

**Если элемент является структурой без какого-либо поля, то (наличие аллокатор не должен занимать в поле указателя на элементы списка.** Это означает, что при выделении узла листа, но по идее без выделения дополнительной операционной памяти, X, потом на узел под лист X, взятого из под памяти X до утечек памяти X, никогда не хватить (кроме требований к Allocator).

Все методы листа должны быть строго безопасны относительно исключений. Это относится, что при выбросе исключения при копировании операции содержимого X, независимо от того, где и на какой момент X было выполнено функцией сложения, X, конструктор или оператор присваивания должны либо полностью завершиться, либо оставить лист X, состоянии до начала выполнения (кроме требований к Allocator).

Проверьте себя: выполните последовательность из случайных добавлений/удалений элементов из листа с StackAllocator, а std::list со стандартным аллокатором.

Как ваш собственный List, так и std::list должны показывать более высокую производительность с List<int, StackAllocator<int, 100000>>, а вот от StackAllocator ум конечно же требует произвольного количества операций добавления/удаления, ваш код будет проверяться путем сравнения производительности к времени стандартного аллокатора, приорирован по времени, а не должен по подобных.
